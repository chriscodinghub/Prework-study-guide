HTML

    Definitions

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • If three elements are contained within a body element, they are then known as children and the body is known as parent.
  • An element referse to the entire element from opening tag to closing tag.
  • A tag refers to what is contained between the angle brackets only.
  • h1 element begins a boldened header.
  • p element is for block text such as paragraphs.
  • ul element is for unordered lists (bullet points) followed by li element (listed items).
  • ol element is for ordered lists (numbered points) followed by li element (listed items).
  • img element contains information displayed on page with sub elements, src attribute is the most important as it defines the location of the image, alt attribute gives a written description of the image, width and height provide a size value for the image.
  • br element created a line break between two blocks of content.
  • a element (which stands for anchor) creates a link to the same webpage or other webpages. href used with a element will link the source webpage and allow you to create a text hyperlink.
  • Section elements are sematic elements that define a section in a document that contains thematic content

CSS

  • For external CSS stylesheet you must imput a link rel="stylesheet" href="./assets/style.css"
  • rel attribute specifies the relationship between the current doc and the linked doc
  • href attribute specifies the location (URL) of the external resource. In this case the URL was assets/style.css
  • The . before card designates this as a class selector. any HTML elements assigned to the class="card" will gain these property assignments from the CSS
  • The first declaration is width
  • The margin property assigned 40px to the top and bottom, and the auto value to the left and right
  • The boarder property assigns a border with a width of 5px and a hex # color assignment
  • The box-shadow property adds a shadow effect with a designated X and Y offset relative to the element and can have a hex # assigned

Git

    Definitions
  • SSH - Secure Shell.
  • MVP - Minimum Value Product - the first iteration of a product w/o extra features.
  • Repository - A folder that holds all files and branches.
  • Branch - A workspace you can make changes to w/o affecting the live application.
  • Markdown (.md) - The way content files are formatted in github (easily converts to HTML)
  • Commit Changes - Saving a new version of a file your working on. Whenever you add, edit, or delete a file you are making a commit, also adds a change to the branch
  • Pull Request (PR) - Asking for approval on changes made to a branch to merge it into the main branch. A way to discuss w/ peers prior to merge.
  • Merge - When PR is approved, changes to branch are "Pulled in" or merged into the main branch. (changes are published to live application)
  • Issue - Way to report new tasks or context fixes. Activly tracks progress on project board. Open Issues for new tasks and close once finished.
  • Acceptance Criteria - Requirements that must be met for us to be complete. Definition of "done".


    • Commands
  • pwd - Print workking dirrectory, shows current dirrectory
  • ls - Shows list of files and folders in current dirrectory
  • cd - navigates between dirrectories
  • cd .. - navigates back one level at a time
  • mkdir - Makes new dirrectory
  • touch - creates new file inside current dirrectory
  • start - opens specific file
  • git proceeds all git commands
  • checkout - moves working branch to a new branch
  • -b flag creates a new branch
  • Example git checkout -b "new branch name" will create an identical branch with a new title
  • git status - checks if new branch is up to date
  • git add -A - add command adds modifications to staging area, -A flag adds to all files
  • git commit -m - saves all changes and allows notes to be attached by coder
  • git pull origin main - checks if the code is current
  • git push origin "name of current branch" - sends new changes to github repo

JavaScript

  • Control flow - The order in which a computer executes code in a script. In JavaScript, code runs from first statement at the top of the page, down to the last statement at the bottom of the page.
  • Variable item - To declare a specific variable item with only one data item.
  • Example code - var topic = "HTML";
  • Array - To create an array, we declare a variable in the same way that we declared a variable with only one data item. However, instead of assigning the variable to a single value, we assign the variable to a group of values.
  • Example code - var shapes = ["triangle", "square", "pentagon", "circle"];
  • JS index always start with a value of 0.
